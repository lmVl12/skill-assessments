## 🏗 Repository Structure

To ensure clarity and ease of review, each project folder follows a standardized format:
* **README.md** — The original problem statement, technical requirements, and guidelines provided by the authors.
* **Raw data files** — The original problem statement, technical requirements, and guidelines provided by the authors.
* **NOTES.md** — My detailed commentary on the solution, methodology descriptions, and direct links to rendered reports.
* **Solution Files** — Source code (`.Rmd`, `.ipynb`, `.R`) and generated outputs (`.html`, `.pdf`).

## 🛠 Technical Notes

> [!IMPORTANT]
> **Path Integrity:** This repository relies on relative file paths. For the code to execute correctly, please clone the entire repository. Opening individual scripts in isolation may result in data-loading errors.

* **Environment:** All code has been tested and is compatible with R 4.5.3 and Python 3.14. Specific dependencies and library versions are documented within the respective `NOTES.md` files.
* **Interactive Reports:** For tasks rendered in HTML, direct links for quick viewing are available in the status table above and within the `NOTES.md` files.
